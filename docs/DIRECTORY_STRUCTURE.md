## Technology Stack Summary

### Frontend
- **Language**: TypeScript with React
- **Build**: Webpack for bundling
- **Styling**: Material-UI components
- **Development**: Express dev server with mock API

### Backend (upstream eclipse/openvsx)
- **Framework**: Spring Boot
- **Database**: PostgreSQL (primary data)
- **Search**: Elasticsearch (indexing)
- **Cache**: Redis cluster (sessions + rate limiting)
- **Task Scheduling**: JobRunr (async jobs)

### Infrastructure
- **Containerization**: Docker multi-stage builds
- **Orchestration**: Kubernetes via OpenShift (OKD)
- **Package Manager**: Helm for deployment
- **CI/CD**: Jenkins with Groovy DSL

### Security
- **Authentication**: OAuth2 with Eclipse Foundation
- **Scanning**: ClamAV (antivirus), YARA (patterns), Gitleaks (secrets)
- **Rate Limiting**: Bucket4j with Redis backend
- **HTTPS**: TLS at OpenShift Route level

### Observability
- **Metrics**: Prometheus + Grafana
- **Logs**: Structured JSON via Logback
- **Traces**: OpenTelemetry (1% sampling)
- **Uptime**: Better Stack monitoring

## Configuration Precedence

Settings are loaded in this order (later overrides earlier):

1. **application.yml** (base configuration in repo)
2. **DEPLOYMENT_CONFIG** environment variable (pointed by Kubernetes ConfigMap)
3. **System properties** (set via JVM args in Helm values)
4. **Environment variables** (set in pod spec)
